Read this first

Add an in-app static cheatsheet page the user can open to remember what the claude CLI can do and how login works. It is a new Activity ('guide') rendered by a GuideView that reads a curated, hand-maintained data module and shows four searchable, grouped sections with a category jump-nav. Deliberately not auto-updating (cost decision) — all content lives in one typed file and is edited by hand.

Key decision: mirror the existing SkillsBrowser pattern (search box + grouped layout + empty state) rather than markdown+react-markdown (weaker search/jump) or Skills-style cards (command lists read worse as cards). Chosen approach A from the spec.

Task 2 — reference/guide.ts + tests (TDD)

Independent of T1. Write the test (RED) then the module (GREEN). Full code lives in the draft markdown next to this file; the shape is:

export interface GuideEntry { command: string; desc: string; example?: string }
export interface GuideSection { id: string; title: string; entries: GuideEntry[] }
export const GUIDE: GuideSection[]            // 4 sections: cli, auth, slash, deck
export function filterGuide(query: string): GuideSection[]
//  q = query.trim().toLowerCase(); '' → GUIDE
//  per section: keep entries whose command/desc/example includes q;
//  if none but title includes q → keep whole section; else drop section.
//  Immutable: returns new section objects ({...section, entries}).

Tests cover: unique ids; every section ≥1 entry; every entry has command+desc; the four ids = ['auth','cli','deck','slash']; filterGuide('') & ' ' → all; match by command/desc/example (case-insensitive); zero-match sections dropped ('push-to-talk' → ['deck']); no-match → []; returned sections always ≥1 entry.
